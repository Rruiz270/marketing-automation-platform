# 🌐 Desenvolvimento na Nuvem com Gitpod

## 🚀 Acesso Instantâneo

Clique no botão abaixo para abrir um ambiente de desenvolvimento completo:

[![Open in Gitpod](https://gitpod.io/button/open-in-gitpod.svg)](https://gitpod.io/#https://github.com/Rruiz270/marketing-automation-platform)

## ✨ O que o Gitpod oferece:

### ✅ Ambiente Completo:
- VS Code no navegador
- Terminal Linux
- Node.js e npm pré-instalados
- Git configurado
- Extensions úteis

### ✅ Servidor Automático:
- Inicia `npm run dev` automaticamente
- Porta 3000 exposta publicamente
- URL acessível de qualquer lugar
- Hot reload funcionando

### ✅ Vantagens:
- Sem configuração local
- Funciona em qualquer computador
- Compartilhável com equipe
- Gratuito para uso público

## 🎯 Como usar:

1. **Clique no link do Gitpod acima**
2. **Faça login com GitHub**
3. **Aguarde o ambiente carregar (1-2 minutos)**
4. **O servidor iniciará automaticamente**
5. **Clique na notificação de "Port 3000" para abrir**

## 💻 Comandos no Gitpod:

```bash
# O ambiente já faz isso automaticamente:
npm install
npm run dev

# Mas você pode usar todos os comandos normais:
git status
git add .
git commit -m "sua mensagem"
git push origin development
```

## 🔧 Alternativas de Acesso:

### Opção 1: Link Direto
https://gitpod.io/#https://github.com/Rruiz270/marketing-automation-platform

### Opção 2: Pelo GitHub
1. Vá para seu repositório no GitHub
2. Mude a URL de `github.com` para `gitpod.io/#https://github.com`
3. Pressione Enter

### Opção 3: Extension do Browser
Instale a extensão Gitpod no Chrome/Firefox para um botão direto no GitHub.

## 🎪 Exemplo de Workflow:

1. **Abrir Gitpod**
2. **Editar arquivos no VS Code**
3. **Ver mudanças instantâneas no preview**
4. **Commit pelo terminal ou VS Code**
5. **Push para deployment automático**

## 💡 Dicas:

- O workspace fica ativo por algumas horas
- Você pode ter múltiplos workspaces
- Ideal para desenvolvimento colaborativo
- Perfeito quando o ambiente local tem problemas
